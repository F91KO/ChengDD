# 程哒哒角色泳道图

## 1. 文档说明

- 文档用途：从角色视角梳理平台、商家、消费者之间的职责边界与协作流程。
- 适用场景：需求评审、职责划分、接口归属判断、页面权限讨论。
- 当前范围：基于现有 PRD 输出业务侧泳道图，不涉及技术调用时序。

## 2. 商家入驻与开通泳道图

```mermaid
flowchart LR
    subgraph Merchant["商家"]
        M1["注册账号"]
        M2["提交主体资料与经营信息"]
        M3["补充资料"]
        M4["配置小程序接入参数"]
        M5["选择首页模板与分类模板"]
        M6["发起一键开通"]
    end

    subgraph Platform["平台"]
        P1["处理资料"]
        P2["返回驳回或通过结果"]
        P3["执行开通校验"]
        P4["初始化店铺、首页、分类"]
        P5["生成首版发布任务"]
        P6["返回发布结果"]
    end

    M1 --> M2 --> P1
    P1 --> P2
    P2 -->|驳回| M3
    M3 --> M2
    P2 -->|通过| M4
    M4 --> M5 --> M6 --> P3
    P3 --> P4 --> P5 --> P6
```

## 3. 首页装修与发布泳道图

```mermaid
flowchart LR
    subgraph Merchant["商家"]
        M1["进入装修中心"]
        M2["选择首页风格"]
        M3["配置首页模块"]
        M4["预览与调整"]
        M5["提交发布"]
    end

    subgraph Platform["平台"]
        P1["提供首页模板与模块预置"]
        P2["校验配置完整性"]
        P3["生成装修版本"]
        P4["发布到目标小程序"]
    end

    subgraph Consumer["消费者"]
        C1["访问首页"]
        C2["查看生效后的首页内容"]
    end

    M1 --> P1
    P1 --> M2
    M2 --> M3 --> M4 --> M5
    M5 --> P2
    P2 -->|通过| P3 --> P4 --> C1 --> C2
    P2 -->|不通过| M4
```
