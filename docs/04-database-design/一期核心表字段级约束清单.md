# 程哒哒一期核心表字段级约束清单

## 1. 文档说明

- 本文用于把一期核心表的字段约束收口到“可以直接指导 DDL 与后端校验”的程度。
- 约束分为两层：
  - 数据库层：`NOT NULL`、默认值、唯一索引、普通索引。
  - 应用层：枚举校验、跨字段校验、状态流转校验、不可逆更新限制。
- 适用范围：一期主链路核心表，不覆盖所有低优先级报表与营销扩展表。

## 2. 通用字段约束

### 2.1 所有核心表通用字段

| 字段 | 约束 | 默认值 | 更新规则 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | 必填 | 无 | 仅创建时写入 | 应用侧发号，禁止更新 |
| `created_by` | 可空 | `NULL` | 仅创建时写入 | 系统任务可为空 |
| `updated_by` | 可空 | `NULL` | 每次更新可写 | 保存最近一次修改人 |
| `created_at` | 必填 | `CURRENT_TIMESTAMP` | 禁止业务更新 | 数据库自动写入 |
| `updated_at` | 必填 | `CURRENT_TIMESTAMP` | 每次更新自动改写 | 数据库自动维护 |
| `deleted` | 必填 | `0` | 仅软删除操作可改 | 只允许 `0 -> 1` |
| `version` | 必填 | `0` | 每次更新递增 | 用于乐观锁 |

### 2.2 通用规则

- MySQL 5.7 不依赖 `CHECK` 约束，范围校验主要由应用层保证。
- 所有唯一索引均默认带 `deleted`，除非字段语义上不允许复用。
- 所有状态字段只允许按状态机流转，不允许随意回写历史状态。
- 所有业务单号字段创建后不可更新。

## 7. 建议直接反映到正式迁移脚本的调整

1. `cdd_merchant_application` 的 `legal_person_name`、`business_category`、`license_file_url` 建议改为 `NOT NULL`，与商家入驻接口必填保持一致。
2. `cdd_order_refund_record.third_party_refund_no` 建议补唯一索引，避免第三方退款回调重复入账。
3. `cdd_release_task_detail` 建议补 `task_id + step_code + deleted` 唯一索引，避免一个任务重复写同一步骤。
4. `cdd_merchant_profile.current_application_id` 建议补普通索引，便于平台按申请追溯商家主体。
5. `cdd_auth_account.account_type` 建议补普通索引，便于平台账号分类型查询。
