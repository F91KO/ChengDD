# 程哒哒核心表索引与唯一约束设计

## 1. 文档说明

- 文档用途：明确核心表的主键、唯一约束、普通索引、联合索引设计建议。
- 当前阶段：逻辑索引设计，用于支撑后续建表 SQL 与性能评审。
- 适用对象：后端研发、DBA、架构、测试。

## 2. 设计原则

- 主键统一使用 `id bigint`
- 唯一约束优先保障业务唯一性，不滥用唯一索引
- 高频查询条件优先建立单列或联合索引
- 联合索引遵循最左匹配原则
- 低区分度字段如 `status` 不单独大量建索引，应与高区分度字段组合使用
- `json` 字段不作为核心过滤索引字段
- 逻辑删除场景下，唯一约束应考虑 `deleted`
