# 程哒哒命名规范

## 1. 文档说明

- 文档用途：统一项目中的服务、模块、代码、数据库、接口、缓存、配置项命名规则。
- 适用范围：产品文档、技术设计、研发实现、测试联调、部署交付。
- 目标：降低沟通成本，避免同一对象在不同层出现多套叫法。

## 2. 总体命名原则

- 全项目统一使用 `cdd` 作为系统前缀。
- 命名优先表达业务语义，不使用含糊缩写。
- 同一层级采用同一种风格，不混用多种命名格式。
- 名称应可扩展，避免写死当前实现方式或单一渠道。
- 对外接口名、数据库字段名、配置项名、缓存 key 命名要能直接看出业务归属。

## 3. 服务命名规范

### 3.1 规则

- 服务统一采用：`cdd-{domain}-service`
- 网关统一采用：`cdd-gateway`
- 公共能力模块不使用 `service` 后缀
- 支付、API、公共包等模块按职责单独命名

### 3.2 当前推荐服务名

- `cdd-gateway`
- `cdd-auth-service`
- `cdd-merchant-service`
- `cdd-decoration-service`
- `cdd-product-service`
- `cdd-order-service`
- `cdd-marketing-service`
- `cdd-release-service`
- `cdd-report-service`
- `cdd-config-service`

### 3.3 不推荐命名

- `merchant-center`
- `system-service`
- `base-service`
- `common-service`
- `catalog-center`
- `release-center-service`

原因：
- 语义过泛，后期职责容易膨胀。

## 4. Maven 模块命名规范

### 4.1 规则

- 父工程：`cdd-parent`
- 公共模块：`cdd-common-*`
- 协议模块：`cdd-api-*`
- 支付能力模块：`cdd-pay-*`
- 服务实现模块：`cdd-*-service`
- 工具模块：`cdd-*-migration`

### 4.2 当前推荐模块

- `cdd-parent`
- `cdd-common-core`
- `cdd-common-db`
- `cdd-common-redis`
- `cdd-common-security`
- `cdd-common-web`
- `cdd-db-migration`
- `cdd-api-auth`
- `cdd-api-merchant`
- `cdd-api-decoration`
- `cdd-api-product`
- `cdd-api-order`
- `cdd-api-marketing`
- `cdd-api-release`
- `cdd-api-report`
- `cdd-api-config`
- `cdd-api-pay`
- `cdd-pay-core`

### 4.3 物理目录命名与职责规范

#### 规则

- 仓库根目录只放跨模块共享内容，不直接放业务 Java 模块源码。
- 所有 Java 模块统一放在 `cdd-parent/` 目录下。
- 仓库级配置目录统一采用 `config/{purpose}/` 结构。
- 正式数据库迁移脚本统一放在 `db/migration/`。
- 仓库级脚本统一放在 `scripts/{purpose}/`。
- 文档统一放在 `docs/`，不与 Java 模块混放。

#### 当前推荐目录

```text
repo/
├─ cdd-parent/
│  ├─ pom.xml
│  ├─ cdd-common-*
│  ├─ cdd-api-*
│  ├─ cdd-db-migration
│  ├─ cdd-pay-core
│  ├─ cdd-gateway
│  └─ cdd-*-service
├─ config/
│  └─ db-migration/
├─ db/
│  └─ migration/
├─ docs/
└─ scripts/
```

#### 职责约束

- `cdd-parent/`：Java 工程聚合根和全部 Java 模块物理目录。
- `config/db-migration/`：数据库迁移执行所需的外部配置，不放业务服务日常运行配置。
- `db/migration/`：Flyway 正式迁移脚本目录，只增不改历史版本。
- `scripts/validation/`：校验、验收、质量门禁脚本。
- `scripts/db/`：数据库迁移、初始化、运维辅助脚本。
- `docs/`：设计、落地、交付类文档。

## 5. Java 包命名规范

### 5.1 规则

- 根包统一采用：`com.cdd`
- 服务包统一采用：`com.cdd.{domain}`
- 包名全部小写
- 不使用中文、下划线、大写字母

### 5.2 推荐结构

```text
com.cdd.product
├─ controller
├─ application
├─ domain
├─ infrastructure
├─ rpc
├─ convert
├─ model
│  ├─ dto
│  ├─ vo
│  ├─ command
│  └─ query
└─ config
```

### 5.3 示例

- `com.cdd.order.controller`
- `com.cdd.order.application`
- `com.cdd.pay.core.wechat`
- `com.cdd.common.db.config`

## 6. Java 类命名规范

### 6.1 基础规则

- 类名采用 `UpperCamelCase`
- 方法名、变量名采用 `lowerCamelCase`
- 常量采用 `UPPER_SNAKE_CASE`
- 布尔字段优先使用 `is` / `has` / `can` 前缀

### 6.2 后缀规范

- Controller：`*Controller`
- Application Service：`*AppService`
- Domain Service：`*DomainService`
- Manager：`*Manager`
- Repository：`*Repository`
- Mapper：`*Mapper`
- Entity：`*Entity`
- DTO：`*DTO`
- VO：`*VO`
- Command：`*Command`
- Query：`*Query`
- Converter：`*Converter`
- Enum：`*Enum`
- Config：`*Config`

### 6.3 示例

- `ProductController`
- `CreateOrderCommand`
- `MerchantApplicationDTO`
- `PayStatusEnum`
- `WechatPayClient`

## 7. 数据库表命名规范

### 7.1 规则

- 表名统一采用 `snake_case`
- 不加 `t_` 前缀
- 表名统一采用：`cdd_{domain}_{object}`
- `{domain}` 必须与服务域命名保持一致
- 单词使用完整业务语义，不使用难理解缩写

### 7.2 推荐方式

- 商家域：`cdd_merchant_*`
- 店铺装修域：`cdd_decoration_*` / `cdd_store_*`
- 商品域：`cdd_product_*`
- 订单域：`cdd_order_*`
- 发布域：`cdd_release_*`
- 报表域：`cdd_report_*` / `cdd_metric_*`

### 7.3 示例

- `cdd_merchant_application`
- `cdd_merchant_store`
- `cdd_decoration_home_config`
- `cdd_product_spu`
- `cdd_product_sku`
- `cdd_product_stock`
- `cdd_product_category_template`
- `cdd_order_info`
- `cdd_order_cart_item`
- `cdd_release_task`

## 8. 数据库字段命名规范

### 8.1 规则

- 字段统一采用 `snake_case`
- 主键统一优先使用 `id` 或 `{object}_id`
- 时间字段统一使用 `_at` 结尾
- 状态字段统一使用 `status`
- 布尔字段统一使用 `is_` / `has_` / `can_`

### 8.2 推荐公共字段

- `id`
- `merchant_id`
- `store_id`
- `mini_program_id`
- `created_at`
- `updated_at`
- `created_by`
- `updated_by`
- `deleted`
- `version`

### 8.3 示例

- `product_id`
- `sku_id`
- `pay_status`
- `release_status`
- `is_enabled`
- `has_refund`

## 9. 接口路径命名规范

### 9.1 规则

- 接口路径统一采用小写 + 中划线或复数名词
- 不在路径中暴露实现细节
- 平台端统一前缀：`/platform`
- 商家端统一前缀：`/merchant`
- 消费者端统一前缀：`/miniapp`

### 9.2 示例

- `/platform/merchants`
- `/platform/releases/tasks`
- `/merchant/products`
- `/merchant/categories/{category_id}/attributes`
- `/miniapp/home`
- `/miniapp/orders`

## 10. Redis Key 命名规范

### 10.1 规则

- 统一采用冒号分隔
- 结构建议：`cdd:{env}:{domain}:{biz}:{identifier}`
- 避免直接使用自然语言
- key 中不放大段文本

### 10.2 示例

- `cdd:prod:auth:token:user_1001`
- `cdd:prod:product:stock:sku_2001`
- `cdd:prod:order:cart:user_3001`
- `cdd:prod:release:lock:task_4001`
- `cdd:test:home:config:store_5001`

## 11. Redisson 锁命名规范

### 11.1 规则

- 统一采用：`cdd:lock:{domain}:{action}:{id}`
- 锁名称要能直观看出锁对象和锁动作

### 11.2 示例

- `cdd:lock:release:publish:task_1001`
- `cdd:lock:product:stock:sku_2001`
- `cdd:lock:order:create:user_3001`

## 12. 配置项命名规范

### 12.1 规则

- Spring 配置统一采用 `kebab-case`
- 自定义配置统一采用 `cdd.{domain}.{item}` 结构
- 开关类配置统一使用正向命名，不用双重否定
- 环境名统一使用：`local`、`dev`、`test`、`prod`
- 可启动模块本地配置文件统一采用：`application.yaml`、`application-{env}.yaml`
- 后续接入 Nacos 后，服务配置 `dataId` 统一采用：`{service-name}-{env}.yaml`
- 后续接入 Nacos 后，共享配置 `dataId` 统一采用：`cdd-common-{env}.yaml`
- 运行时环境与配置模式键统一保留：`cdd.runtime.env`、`cdd.runtime.config-mode`
- 数据库迁移配置文件固定为 `config/db-migration/application-db-migration.yml`，不并入 Nacos 命名空间

### 12.2 示例

- `cdd.product.stock.lock-timeout-ms`
- `cdd.release.gray.enabled`
- `cdd.pay.wechat.notify-url`
- `cdd.home.fresh-retail.default-enabled`
- `cdd.runtime.env`
- `cdd.runtime.config-mode`
- `cdd-order-service-dev.yaml`
- `cdd-common-test.yaml`

## 13. 枚举值命名规范

### 13.1 规则

- 枚举类采用 `*Enum`
- 枚举值统一采用大写下划线
- 语义正交、避免同义词重复

### 13.2 示例

- `ENABLED`
- `DISABLED`
- `PENDING_PAY`
- `PENDING_SHIP`
- `REFUNDING`
- `ROLLBACKED`

## 14. MQ Topic / Event 命名规范

### 14.1 规则

- 统一采用：`cdd.{domain}.{event}`
- 事件名使用过去式或结果态，表达“已发生”

### 14.2 示例

- `cdd.order.created`
- `cdd.order.paid`
- `cdd.order.refunded`
- `cdd.product.published`
- `cdd.release.finished`

## 15. 文件与文档命名规范

### 15.1 代码相关

- 配置文件：`application.yaml`、`application-{env}.yaml`
- Nacos 配置：`{service-name}-{env}.yaml`、`cdd-common-{env}.yaml`
- SQL 文件：`V{version}__{desc}.sql`
- Shell 脚本：`start-cdd-order-service.sh`

### 15.2 文档相关

- 中文文档可使用中文命名，但尽量语义明确
- 技术设计文档建议同层级保持统一风格

示例：
- `技术架构图.md`
- `数据库设计草案.md`
- `服务与表关系矩阵.md`

## 16. 当前推荐命名总结

- 服务：`cdd-{domain}-service`
- 公共包：`cdd-common-*`
- 协议包：`cdd-api-*`
- 支付模块：`cdd-api-pay`、`cdd-pay-core`
- 根包：`com.cdd.{domain}`
- 表名：`cdd_{domain}_{object}`
- 字段名：`snake_case`
- 类名：`UpperCamelCase`
- 方法名：`lowerCamelCase`
- Redis Key：`cdd:{env}:{domain}:{biz}:{id}`
- 锁名：`cdd:lock:{domain}:{action}:{id}`
