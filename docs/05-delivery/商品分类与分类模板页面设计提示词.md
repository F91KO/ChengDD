# 商品分类与分类模板页面设计提示词

## 使用说明

本提示词用于生成以下页面方案：

- 商家端：商品分类管理页
- 平台端：分类模板管理页

适用对象：

- Figma MCP
- 页面设计模型
- 前端页面生成模型

设计基线：

- 业务场景为生鲜零售多商户平台
- 风格参考头部生鲜即时零售与社区到家平台
- 文案统一使用中文
- 初始化分类数据需具备真实零售经营感，不使用“分类A / 分类B”这类占位内容

## 提示词正文

```text
请为“ChengDD 生鲜零售平台”设计两张后台页面，输出高保真页面方案与组件结构说明，语言使用中文。

项目背景：
这是一个支持多商户的小程序商城平台，平台负责模板、分类模板、发布治理和配置能力，商家负责自己的商品、分类、订单和运营内容。当前要补齐“商品分类”和“分类模板”能力。

设计目标：
1. 页面必须体现“生鲜零售”行业特征，参考头部即时零售与社区到家业态的商品组织方式。
2. 视觉风格要现代、明快、专业，偏零售运营后台，不要做成传统 ERP，也不要做成空泛通用管理台。
3. 页面必须适合桌面端后台使用，信息密度中高，但层次清晰。
4. 所有文案使用中文。
5. 页面中的数据、分类结构、模板名称都要有真实业务感，不能是泛泛的“分类A/模板B”。

需要输出的页面：
1. 商家端“商品分类管理”页面
2. 平台端“分类模板管理”页面

一、商品分类管理页要求
页面目标：
- 商家查看当前分类树
- 从平台分类模板初始化分类
- 新增分类
- 编辑分类
- 查看分类层级、排序、启用状态、展示状态
- 明确哪些分类来自模板，哪些是商家自定义扩展

页面结构建议：
1. 页面头部
   - 标题：商品分类
   - 副标题：管理门店商品分类结构，支持从平台模板初始化并按经营需要调整
   - 右侧主按钮：新增分类
   - 次按钮：从模板初始化
2. 顶部信息卡
   - 分类总数
   - 启用分类数
   - 前台展示分类数
   - 最近更新时间
3. 左侧主区域：分类树 / 分层列表
   - 支持 3 级分类展示
   - 每个分类节点展示：
     - 分类名称
     - 层级
     - 排序值
     - 启用/停用状态
     - 前台展示/隐藏状态
     - 来源标签：模板同步 / 商家新增
   - 节点操作：
     - 新增子分类
     - 编辑
     - 停用/启用
4. 右侧详情面板
   - 当前分类信息
   - 父分类
   - 来源模板
   - 分类说明
   - 关联商品数
   - 最近修改时间
   - 可执行操作按钮
5. 初始化弹窗
   - 展示平台可用分类模板列表
   - 支持选择模板后预览分类结构
   - 强调“初始化后会生成商家自己的分类树，可继续调整”

二、分类模板管理页要求
页面目标：
- 平台查看分类模板列表
- 创建分类模板
- 查看模板详情
- 维护模板分类结构
- 标识模板版本、行业、状态、适用商家数
- 支持“推荐模板 / 停用模板 / 历史版本”信息表达

页面结构建议：
1. 页面头部
   - 标题：分类模板管理
   - 副标题：维护平台标准分类模板，供商家初始化商品分类树
   - 主按钮：新建模板
2. 顶部统计卡
   - 模板总数
   - 启用中模板
   - 推荐模板
   - 已接入商家数
3. 主列表区域
   - 模板名称
   - 适用行业
   - 模板版本
   - 最大层级
   - 状态
   - 最近发布时间
   - 已使用商家数
   - 操作：查看详情 / 编辑 / 复制 / 停用
4. 模板详情页或侧边详情面板
   - 模板基础信息
   - 分类树结构预览
   - 模板说明
   - 适用品类
   - 运营建议
   - 适用门店类型
   - 示例商家
5. 模板编辑区域
   - 支持维护分类树节点
   - 每个节点展示名称、层级、排序、是否默认展示、建议属性数

三、分类初始化数据要求
初始化数据必须参考头部即时零售 / 生鲜超市业态，分类要真实、接地气，适合社区生鲜、电商到家、小程序商城场景。

请提供一套真实感较强的分类模板示例，至少包含 3 级分类，建议覆盖以下方向：
1. 蔬菜豆制品
   - 叶菜类
   - 根茎类
   - 菌菇类
   - 豆制品
2. 水果鲜切
   - 应季水果
   - 进口水果
   - 鲜切果盒
3. 肉禽蛋品
   - 猪肉
   - 牛羊肉
   - 鸡鸭禽类
   - 蛋品
4. 海鲜水产
   - 活鲜水产
   - 冷冻海鲜
   - 贝类虾蟹
5. 乳品烘焙
   - 鲜奶酸奶
   - 面包蛋糕
   - 早餐轻食
6. 酒水饮料
   - 即饮饮料
   - 茶咖冲调
   - 啤酒葡萄酒
7. 粮油调味
   - 大米杂粮
   - 食用油
   - 调味酱料
8. 方便速食
   - 速冻面点
   - 预制菜
   - 即食熟食
9. 休闲零食
   - 坚果果干
   - 膨化零食
   - 糖巧饼干
10. 日清百货
   - 厨房用品
   - 清洁纸品
   - 日化洗护

请同时给出 2 套模板风格：
1. 品质精选生鲜模板
   - 更强调“餐桌场景、品质生鲜、鲜切与半成品、即时零售”
2. 社区民生到家模板
   - 更强调“社区到家、日常高频补货、民生商品结构”

四、视觉风格要求
1. 配色参考高品质生鲜零售后台：
   - 主色可用清新绿色、橙红点缀、米白/浅灰底
   - 不要紫色科技风
   - 不要纯黑深色后台
2. 版式要有零售感和商品组织感，不只是传统表格
3. 分类节点可适度加入图标、标签、层级线、状态色块
4. 模板卡片要有“行业模板”“推荐模板”“版本”这类明显识别
5. 字体和间距要有呼吸感，但不能过度空旷
6. 页面应兼顾信息密度与可读性，适合运营人员长时间使用

五、输出要求
请输出：
1. 两张页面的完整高保真设计说明
2. 页面布局结构
3. 核心组件清单
4. 关键中文文案
5. 示例分类初始化数据
6. 设计风格说明
7. 如果合适，可补充每张页面的交互状态：
   - 空状态
   - 初始化成功状态
   - 无分类状态
   - 编辑中状态
   - 初始化覆盖提醒弹窗
```
