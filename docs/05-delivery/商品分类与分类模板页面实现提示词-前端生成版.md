# 商品分类与分类模板页面实现提示词 前端生成版

## 用途

本提示词适用于：

- 前端代码生成模型
- 页面联调生成模型
- 基于现有仓库风格补齐页面实现

## 提示词

```text
你现在在 ChengDD 项目内工作，请实现“商品分类管理页”和“分类模板管理页”的前端页面，要求与现有商家后台风格一致，中文提示信息，走真实接口，不允许使用 mock 数据。

项目事实：
- 前端项目：cdd-frontend
- 技术栈：Vue 3 + Vite + TypeScript + Pinia + Vue Router + Axios + CSS Modules
- 当前已有页面：工作台、商品管理、订单、售后、配置中心、发布治理
- 风格需保持现有页面一致，不重做设计系统

接口背景：
- 商品分类相关接口：
  - GET /api/product/categories
  - POST /api/product/categories/init
  - POST /api/product/categories
  - PUT /api/product/categories/{category_id}
- 分类模板相关接口：
  - GET /api/product/category-templates
  - POST /api/product/category-templates

实现目标：
1. 新增“商品分类管理页”
2. 新增“分类模板管理页”
3. 菜单与路由补齐入口
4. 页面必须基于真实接口渲染
5. 商品管理页中的分类显示应尽量展示分类名称，而不是只有分类 ID

商品分类管理页要求：
- 页面头部包含：
  - 标题：商品分类
  - 副标题：管理当前商家的分类树，可从平台模板初始化
  - 按钮：新增分类、从模板初始化、刷新
- 页面主体包含：
  - 顶部统计卡：分类总数、启用分类数、前台展示分类数
  - 分类树/分类列表
  - 右侧详情或编辑面板
- 支持操作：
  - 查询分类列表
  - 查询模板列表
  - 选择模板并初始化分类树
  - 新增分类
  - 编辑分类名称、排序、启用状态、展示状态
- 页面要清晰区分：
  - 模板同步分类
  - 商家自定义分类

分类模板管理页要求：
- 页面头部包含：
  - 标题：分类模板
  - 副标题：查看平台可用分类模板与结构
  - 按钮：新建模板、刷新
- 页面主体包含：
  - 模板统计卡
  - 模板列表
  - 模板详情预览
- 模板详情至少展示：
  - 模板名称
  - 行业
  - 版本
  - 最大层级
  - 状态
  - 模板说明
  - 分类树结构
- 如时间有限，模板创建可以先做基础表单与真实接口接入，不必一次完成复杂模板编辑器

页面风格要求：
- 保持现有 WorkspaceLayout / UiCard / UiButton / UiInput / UiStatePanel / UiTag 组件风格
- 中文文案
- 不要做成粗糙表格堆砌
- 适度增强零售感和层级结构展示

分类模板示例数据理解：
- 页面展示需要适配两套模板风格：
  - 品质精选生鲜模板
  - 社区民生到家模板
- 页面中的模板说明、标签、预览文案要体现出生鲜零售真实经营语境

实现约束：
- 只在现有仓库内修改
- 不要回滚无关改动
- 不要引入新的 mock fallback
- 与现有真实接口保持字段契约一致
- 中文错误提示

验收要求：
1. 页面可访问
2. 页面可读取真实模板和分类数据
3. 页面可发起分类初始化
4. 页面可新增分类、编辑分类
5. 商品管理页中的分类显示同步改善
6. corepack pnpm build 通过

输出要求：
- 直接实现
- 完成后给出：
  - 修改摘要
  - 关键文件
  - 验证命令
  - 剩余风险
```
