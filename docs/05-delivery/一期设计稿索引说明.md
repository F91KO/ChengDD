## 4. 与一期功能映射

### 4.1 已覆盖的核心模块

- 认证登录
- 商家首页工作台
- 商户资料与接入配置
- 商品管理
- 订单管理
- 售后处理
- 发布治理
- 配置中心
- 权限管理
- API 调试

### 4.2 当前最适合优先前端落地的页面

建议按下面顺序进入前端开发：

1. 登录页
2. 商家后台首页
3. 商品管理
4. 订单管理
5. 售后处理
6. 配置中心

原因：
- 这几页最直接承接一期已完成后端能力。
- 能最快形成“可访问、可演示、可联调”的商家后台闭环。

## 5. 当前风格结论

- 主体风格统一，整体符合“一期后台”定位。
- 已体现的设计方向：
  - 深蓝灰主基调
  - 橙色强调按钮和关键状态
  - 卡片化、低噪音
  - 轻量但偏企业后台
- 适合作为一期后台视觉基线继续扩展。

- 补一套桌面端主稿：
  - 登录
  - 首页
  - 商品列表
  - 订单列表

- 前端实现时，不建议一次性铺满全部页面。
- 建议先做“商家后台 MVP”：
  - 登录
  - 首页
  - 商品管理
  - 订单管理
  - 售后处理
