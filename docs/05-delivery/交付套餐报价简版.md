# 程哒哒交付套餐报价简版

适用场景：用于客户初步沟通、商务报价说明、销售同事转发

说明：本文档为“报价单风格 1 页版”，用于说明不同交付套餐的定位与边界，不含具体金额。若进入正式报价阶段，可在此基础上补充报价项、实施周期和付款节点。

## 1. 套餐总览

| 套餐 | 适合客户 | 核心价值 | 推荐结论 |
| --- | --- | --- | --- |
| 轻量版 | 小微企业、初创团队、预算敏感客户 | 先低成本上线，先把开通、发布、数据查看跑起来 | 适合先上再扩 |
| 标准版 | 计划长期运营、需要规范交付的客户 | 环境更完整，发布更稳，后续扩展更顺 | 默认推荐 |
| 高可用版 | 政企客户、中大型客户、强监管客户 | 稳定性更高，审计更完整，适合长期生产运行 | 高规格项目推荐 |

## 2. 套餐说明

### 2.1 轻量版

- 适用客户：预算有限，希望先尽快把系统上线的小微企业客户。
- 核心价值：用更低的基础设施成本完成首期交付，支持商家接入、小程序发布、基础数据查看。
- 部署建议：`1 套生产环境 + 1 套测试/预发合并环境`，优先复用客户现有云资源和托管服务。
- CI/CD 服务：默认不强制，可按需提供轻量流水线模板或代建服务。
- 客户需准备：云主机或服务器、数据库、Redis、对象存储、域名证书、小程序账号、支付账号。
- 不含内容：高可用集群、复杂监控平台、专网改造、双活灾备、长期驻场运维。
- 推荐话术：如果客户说“预算有限，先上线再说”，优先推荐轻量版。

### 2.2 标准版

- 适用客户：准备长期运营，希望项目交付规范、环境完整、后续扩展平稳的客户。
- 核心价值：按正式项目方式交付，具备更规范的环境规划、版本发布和运维交接能力。
- 部署建议：`开发 + 测试 + 预发 + 生产` 分环境建设，支持更规范的联调、验收和发布流程。
- CI/CD 服务：建议作为配套项提供，支持标准构建、部署、回滚和发布留痕。
- 客户需准备：多环境资源、数据库与缓存实例、对象存储、日志采集能力、测试和验收人员。
- 不含内容：专网隔离改造、跨地域灾备、强监管审计定制、非标准第三方系统深度集成。
- 推荐话术：如果客户说“系统后面会长期用，不能太随意”，优先推荐标准版。

### 2.3 高可用版

- 适用客户：对稳定性、安全、审计、容灾有明确要求的政企或中大型客户。
- 核心价值：按照长期生产系统标准建设，降低故障风险，提升审计、备份和恢复能力。
- 部署建议：应用多实例、负载均衡、数据库主从或集群、独立日志监控和备份体系。
- CI/CD 服务：建议纳入标准交付范围，支持多环境发布、审批、回滚和审计留痕。
- 客户需准备：多节点资源、网络安全策略、监控与备份平台、运维和安全负责人。
- 不含内容：客户内部制度建设、等保测评实施、第三方安全厂商服务，除非另行约定。
- 推荐话术：如果客户说“稳定性和安全要求很高”，优先推荐高可用版。

## 3. 客户选择建议

1. 如果客户最关心预算，先推轻量版。
2. 如果客户最关心长期使用和规范交付，先推标准版。
3. 如果客户最关心安全、审计和容灾，先推高可用版。
4. 如果客户当前预算有限但未来会扩容，建议先签轻量版，同时约定后续升级到标准版的方式。

## 4. 统一交付边界

1. 三个套餐交付的都是同一套平台能力，差别主要在部署规格、环境完整度和运维保障强度。
2. 轻量版不是功能缩水版，而是基础设施和交付规格更轻。
3. 所有套餐默认基于统一代码主干交付，不为单个商家长期维护独立分叉版本。
4. 若客户需要专网部署、强监管改造、深度二次开发或驻场实施，应单独列项报价。

## 5. 可作为报价单附页的简短说明

可直接附在报价单中的说明文字：

“本项目采用统一代码底座交付，不同套餐的主要差异在于部署规格、环境规划和运维保障级别。若客户当前以快速上线和控制成本为主，建议选择轻量版；若以长期稳定运营为目标，建议选择标准版；若对稳定性、安全和审计有更高要求，建议选择高可用版。后续可根据业务发展按阶段升级部署方案，无需重做核心业务系统。”
