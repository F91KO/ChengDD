# 需求漂移纠偏任务清单

## 1. 目标

- 以需求文档为唯一基线，纠正当前一期实现中已经确认的需求漂移。
- 将“分类、分类模板、商品”设为当前最高优先级，优先保证页面可用性、真实接口联调和核心链路无明显 bug。
- 纠偏范围聚焦 3 类问题：
  - 分类模板页面归属漂移
  - 订单发货模型漂移
  - 订单状态契约漂移
- 同步修正文档与验收口径，避免后续继续以偏移实现扩散。

## 1.1 当前执行状态

- 已完成：
  - 分类模板归属纠偏
  - 商品分类模块收口
  - 商品管理模块收口
  - 商家端分类模板入口下线
- 已落地结果：
  - 商家后台不再承载分类模板管理页
  - 平台侧已承接 `/platform/category-templates` 页面
  - 商家侧仅保留模板查询与分类树初始化使用链路
- 待继续：
  - 订单发货接口纠偏
  - 订单状态契约纠偏
  - 相关验收与交付文档最终同步

## 2. 已确认基线

- 分类模板管理页属于平台后台，不属于商家后台。
- 商家侧仅保留“查看可用平台分类模板并初始化商家分类树”的使用能力。
- 分类、分类模板、商品模块优先于订单纠偏任务执行。
- 这 3 个模块必须满足以下底线：
  - 页面不使用 mock 或内存假数据
  - 主要按钮、表单、分页、搜索、详情、状态操作可走真实接口
  - 无明显死链、空白页、按钮无响应、成功失败提示缺失
- 订单发货必须保持需求口径：
  - 使用发货接口
  - 必填物流公司编码
  - 必填物流公司名称
  - 必填物流单号
  - 仅允许在需求约定状态下发货
- 订单状态口径保持与需求文档一致，不接受当前偏移实现直接转正。

## 3. 任务拆解

### P0-1 商品分类模块收口

- 目标：保证商家侧“商品分类”页面真实可用，核心操作可联调闭环。
- 范围：
  - 分类列表加载、分页、搜索、选中态、批量选择
  - 从平台分类模板初始化分类树
  - 新增一级分类
  - 新增子分类
  - 编辑分类
  - 启用 / 停用 / 展示 / 隐藏等状态操作
  - 页面提示、空态、失败态、刷新动作
- 重点检查：
  - 初始化后数据是否即时刷新
  - 分类树层级、父子关系、排序值是否正确回显
  - 分页、搜索和选择状态是否互相污染
  - 表单校验和错误提示是否完整

验收：
- 商品分类页所有核心操作都走真实接口。
- 无按钮无响应、重复提交、状态未刷新、分页错乱等明显问题。
- 初始化分类树后，可立即查看、编辑和继续新增分类。

### P0-2 分类模板模块收口

- 目标：保证分类模板能力按“平台后台管理、商家侧仅使用”的基线收口。
- 范围：
  - 商家后台移除分类模板管理入口
  - 商家侧仅保留模板查询与初始化使用链路
  - 分类模板管理能力归位到平台后台口径
  - 模板列表、详情、节点预览、创建能力的职责边界梳理
- 重点检查：
  - 商家菜单、路由、页面跳转中是否仍残留分类模板管理入口
  - 商家分类页是否还能正常读取平台模板列表
  - 文档、页面文案、验收口径是否仍把模板管理写成商家能力

验收：
- 商家后台不再承载分类模板管理页。
- 商家分类初始化链路仍然可用。
- 平台后台归属口径在产品、接口、交付文档中一致。

当前状态：
- 已完成。

### P0-3 商品管理模块收口

- 目标：保证商品管理页面在真实接口下可完成商品全生命周期基础操作。
- 范围：
  - 商品列表加载、分页、关键词搜索
  - 新增商品
  - 编辑商品
  - SKU 编辑
  - 上下架
  - 库存调整
  - 详情面板与状态提示
- 重点检查：
  - 商品分类下拉是否可用且数据一致
  - 新增、编辑后列表与详情是否同步刷新
  - SKU 新增删除、价格库存输入、库存调整是否正确提交
  - 搜索、分页、状态切换是否存在数据错位或回显错误

验收：
- 商品管理页所有核心操作都走真实接口。
- 新增、编辑、上下架、库存调整链路可用。
- 列表、详情、统计卡片、分页信息保持一致。

### T1 设计基线冻结

- 基于需求文档冻结本轮纠偏范围与目标口径。
- 明确以下对象的标准定义：
  - 分类模板管理页归属
  - 订单发货接口语义
  - 订单状态枚举
  - 订单状态流转规则
- 输出“需求基线 vs 当前实现 vs 纠偏目标”的统一对照表。

验收：
- 所有纠偏任务都能明确引用同一份基线，不再出现“以当前实现为准”的歧义。

### T2 商家后台分类模板入口下线

- 移除商家后台中的“分类模板”菜单入口。
- 移除商家后台 `/category-templates` 路由。
- 清理商家后台页面、文案、跳转链路中的分类模板管理承载。
- 保留商品分类页中的“从平台分类模板初始化分类树”能力。

验收：
- 商家后台不再出现分类模板管理入口。
- 商品分类页仍可读取平台分类模板列表并执行初始化。

### T3 平台后台分类模板能力归位

- 明确分类模板管理能力归属平台后台。
- 梳理平台后台后续需要承接的能力范围：
  - 模板列表
  - 模板详情
  - 创建模板
  - 编辑模板
  - 复制模板
  - 模板状态变更
- 本轮先完成职责归位和文档口径修正，不要求立即建设平台后台前端页面。

验收：
- 产品文档、接口文档、任务清单中对分类模板归属的描述一致。

### T4 订单发货接口纠偏

- 将当前“履约状态推进”能力恢复为需求定义的“物流发货”能力。
- 前端订单发货交互需改为物流发货表单，至少包含：
  - 物流公司编码
  - 物流公司名称
  - 运单号
- 前端请求地址、请求字段、成功提示、失败提示与需求口径一致。
- 后端请求模型、Controller、应用服务和仓储校验同步调整。

验收：
- 未填写物流字段时无法发货。
- 订单发货成功后可查询到完整物流信息。
- 当前仅支持在需求规定状态发货。

### T5 订单状态契约纠偏

- 梳理需求文档中的订单状态枚举和状态流转。
- 识别当前代码中的状态拆分点：
  - `order_status`
  - `delivery_status`
- 输出纠偏方案，统一对外接口、前端展示、文档与验收口径。
- 如内部实现仍需保留细粒度字段，需增加明确映射规则，但对外口径仍必须与需求一致。

验收：
- 前端页面、接口返回、接口文档中的订单状态命名一致。
- 不再出现对外状态机与需求文档不一致的问题。

### T6 文档与验收材料同步

- 修正以下文档中的滞后口径：
  - `cdd-frontend/README.md`
  - `docs/05-delivery/reports/frontend-acceptance.md`
  - `docs/05-delivery/当前任务收口清单.md`
  - `cdd-frontend/src/pages/LoginPage.vue` 中的旧基线文案
- 补充本轮纠偏后的验收说明、验证步骤和风险提示。

验收：
- 设计文档、接口文档、前端文档、验收文档口径一致。
- Java 版本、真实接口状态、页面归属描述全部更新到当前基线。
