# 骨架验证与验收说明

## 1. 目标

- 对 Java Spring 多模块骨架做可重复执行的边界校验、构建校验、启动校验。
- 将 `#13 feature-skeleton-validation` 从人工验收转为脚本化验收。

## 2. 验收内容

- 模块聚合是否完整
- 模块依赖边界是否正确
- `JDK 17` 基线是否生效
- `cdd-parent` 的 `local / dev / test / prod` profile 是否完整
- Maven `validate / compile / install` 是否通过
- `cdd-gateway`、`cdd-merchant-service` 最小启动是否通过
- `db/migration` 迁移文件是否存在且范围完整
- 迁移配置是否已从根 `pom.xml` 外置到独立 `yml`
- 每个可启动模块的环境配置文件组是否完整

## 3. 本地执行

```bash
bash scripts/validation/validate_backend_skeleton.sh
```

如需指定自定义 Maven settings 或本地仓库：

```bash
export CDD_MAVEN_SETTINGS=/path/to/settings.xml
export CDD_MAVEN_REPO=/path/to/.m2/repository

bash scripts/validation/validate_backend_skeleton.sh
```

## 4. 脚本说明

- `scripts/validation/check_module_boundaries.py`
  - 校验根模块清单
  - 校验根 profile 清单
  - 校验 `common / api / pay-core / gateway / service` 之间的依赖方向
  - 校验可执行模块的 Spring Boot 打包约定
  - 校验 `db/migration` 迁移文件范围
  - 校验每个可启动模块的 `application*.yaml` 文件组存在
  - 校验迁移配置文件存在

- `scripts/validation/validate_backend_skeleton.sh`
  - 统一执行模块边界校验、Maven 构建校验、服务启动冒烟校验

## 5. 测试报告要求

- 无论测试由主 agent 还是测试 sub-agent 执行，完成后都必须输出测试报告。
- 测试报告至少包含以下内容：
  - 测试范围
  - 执行时间
  - 执行命令
  - 通过项
  - 失败项与失败原因
  - 关键日志或错误摘要
  - 是否达到当前任务验收标准
- 若本轮未执行某项计划内测试，报告中必须明确说明未执行原因与风险。

## 6. CI

- 已新增 GitHub Actions：
  - `.github/workflows/backend-skeleton-validation.yml`
- 在 `main` 和 `feature-*` 分支 push、PR 时自动执行。

## 7. 当前通过口径

通过必须同时满足：

- 边界校验通过
- `mvn validate` 通过
- `mvn compile` 通过
- `mvn install` 通过
- `cdd-auth-service` 启动成功
- `cdd-gateway` 启动成功
- `cdd-merchant-service` 在无数据源配置情况下启动成功
- 数据库迁移连接参数位于 `config/db-migration/application-db-migration.yml`
- 每个可启动模块在自己的 `src/main/resources/` 下维护环境配置
