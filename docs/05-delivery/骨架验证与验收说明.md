# 一期自动化测试与验收基线说明

## 1. 目标

- 收口一期核心链路的自动化测试矩阵、执行命令与验收口径。
- 复用已有验证脚本与服务测试结果，统一输出可审计的测试报告。
- 作为 issue #27 的交付基线，支撑本地执行与 CI 对齐。

说明：

- 本文中的“自动化测试”默认指单元测试、集成测试、启动验证、验收脚本中的适用组合，不等于只跑单元测试。

## 2. 前置条件

- JDK：`17`（强制）
- Maven：`3.6.3+`
- 仓库根目录：`/Volumes/workspace/ChengDD`
- 可选环境变量：
  - `CDD_MAVEN_SETTINGS`
  - `CDD_MAVEN_REPO`
  - `CDD_PHASE1_TEST_MODULES`
  - `CDD_PHASE1_REPORT_FILE`

## 3. 一期核心链路测试矩阵

| 层级 | 核心链路 | 默认执行方式 | 验收要点 |
| --- | --- | --- | --- |
| 结构与边界 | 模块聚合、依赖方向、profile、迁移文件连续性、配置文件完整性 | `python3 scripts/validation/check_module_boundaries.py` | 模块边界无违规，配置资产齐全 |
| 构建与启动冒烟 | `validate/compile/install` + `auth/gateway/merchant` 启动 | `bash scripts/validation/validate_backend_skeleton.sh` | 构建通过，关键服务可启动 |
| 核心业务测试 | `common-security/auth/gateway/merchant/product/order/release/config` | `bash scripts/testing/run_phase1_acceptance.sh` | 模块测试报告全量生成且无失败 |
| 报告与验收 | 统一测试报告、达标结论、未执行项风险 | 自动输出到 `docs/05-delivery/reports/` | 报告字段完整、结论可追溯 |

## 4. 本地执行命令

执行一期验收（默认会先跑骨架校验，再跑模块测试，并输出报告）：

```bash
bash scripts/testing/run_phase1_acceptance.sh
```

仅复用已有测试结果并生成报告（不重跑 `mvn test`）：

```bash
bash scripts/testing/run_phase1_acceptance.sh --reuse-existing-results --skip-skeleton
```

仅执行骨架校验：

```bash
bash scripts/validation/validate_backend_skeleton.sh
```

## 5. CI 执行命令

当前 CI 工作流 `.github/workflows/backend-skeleton-validation.yml` 使用命令：

```bash
bash scripts/validation/validate_backend_skeleton.sh
```

当 CI 切换到一期验收全量口径时，命令应调整为：

```bash
bash scripts/testing/run_phase1_acceptance.sh
```

## 6. 测试报告模板与输出

- 模板文件：`docs/05-delivery/测试报告模板.md`
- 默认输出：`docs/05-delivery/reports/phase1-acceptance-latest.md`
- 报告必须包含：
  - 测试范围
  - 执行时间
  - 执行命令
  - 通过项
  - 失败项与失败原因
  - 关键日志或错误摘要
  - 是否达到当前任务验收标准
  - 未执行项与风险
