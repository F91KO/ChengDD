# 商品分类与分类模板页面设计提示词 Figma MCP 版

## 用途

本提示词适用于：

- Figma MCP
- Stitch / 页面设计生成工具
- 高保真后台页面设计生成

目标页面：

- 商家端商品分类管理页
- 平台端分类模板管理页

## 提示词

```text
请为 ChengDD 生鲜零售平台设计两张中文后台页面：
1. 商品分类管理页
2. 分类模板管理页

业务背景：
- 多商户小程序商城平台
- 平台负责模板、分类模板、发布治理、配置能力
- 商家负责商品、分类、订单、经营内容
- 当前要补齐“商品分类”和“分类模板”页面方案

风格要求：
- 参考头部生鲜即时零售与社区到家平台的后台气质
- 清爽、现代、可信、偏运营后台
- 不要紫色科技风
- 不要纯黑深色后台
- 不要传统 ERP 风格大表格堆叠
- 配色建议：清新绿色、橙色点缀、米白或浅灰背景

页面一：商品分类管理
页面目标：
- 查看当前分类树
- 从平台分类模板初始化分类
- 新增分类
- 编辑分类
- 查看分类状态、排序、来源

页面结构：
- 页面头部：标题、副标题、主按钮“新增分类”、次按钮“从模板初始化”
- 顶部统计卡：分类总数、启用分类数、前台展示分类数、最近更新时间
- 主区域左侧：三级分类树
- 主区域右侧：分类详情面板
- 初始化弹窗：模板选择、模板预览、初始化说明

分类节点展示信息：
- 分类名称
- 分类层级
- 排序值
- 启用/停用
- 前台显示/隐藏
- 来源标签：模板同步 / 商家新增

页面二：分类模板管理
页面目标：
- 查看模板列表
- 查看模板详情
- 新建模板
- 编辑模板结构
- 展示模板版本、行业、状态、适用商家数

页面结构：
- 页面头部：标题、副标题、主按钮“新建模板”
- 顶部统计卡：模板总数、启用模板、推荐模板、已接入商家数
- 列表区域：模板名称、行业、版本、最大层级、状态、发布时间、已使用商家数、操作按钮
- 详情面板：模板基础信息、分类树预览、适用品类、运营建议、适用门店类型

分类初始化数据要求：
- 必须参考头部即时零售与社区到家业态
- 必须是生鲜即时零售真实分类，不允许“分类A/分类B”
- 至少 3 级分类
- 需包含以下大类：
  - 蔬菜豆制品
  - 水果鲜切
  - 肉禽蛋品
  - 海鲜水产
  - 乳品烘焙
  - 酒水饮料
  - 粮油调味
  - 方便速食
  - 休闲零食
  - 日清百货

请提供两套模板风格：
1. 品质精选生鲜模板
   - 强调品质生鲜、鲜切、半成品、餐桌场景、即时零售
2. 社区民生到家模板
   - 强调社区到家、民生高频补货、家庭日常采购

输出内容：
- 两张页面的高保真设计说明
- 页面布局结构
- 核心组件清单
- 中文文案建议
- 示例分类初始化数据
- 关键交互状态：
  - 空状态
  - 初始化成功状态
  - 无分类状态
  - 编辑中状态
  - 初始化覆盖提醒弹窗
```
