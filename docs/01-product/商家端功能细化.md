# 程哒哒商家端功能细化说明

## 1. 文档定位

- 文档用途：基于 PRD 拆分商家侧功能，便于产品设计、原型输出、研发排期和客户沟通。
- 适用对象：产品经理、设计师、研发、测试、实施人员、商家侧使用方。
- 对应范围：商家主账号、子账号在后台可见和可操作的全部核心功能。

## 2. 商家端目标

1. 让商家在不改代码的前提下完成小程序接入、开通、发布和日常维护。
2. 让商家保留自己的店铺风格、分类结构、商品内容和经营方式。
3. 让商家可自行维护多个店铺、多个小程序，以及多个子账号。
4. 让商家能通过标准配置、功能开关和定制申请满足差异化需求。
5. 让商家可查看订单、退款、账单和经营数据，但商家收入不计入平台。

## 3. 商家端角色

### 3.1 主账号

- 负责商家整体资料维护、小程序接入、店铺管理、发布管理。
- 负责创建和管理子账号。
- 默认拥有当前商家全部店铺、小程序和数据权限。

### 3.2 子账号

- 由主账号创建和授权。
- 可按店铺、小程序、模块、动作维度获得权限。
- 仅可访问被授权范围内的数据和功能。

## 4. 商家端功能边界

1. 商家端负责商家自有内容维护和业务操作，不负责平台级模板治理。
2. 商家可管理自己的商品分类树和分类属性，分类与属性初始来源于平台模板。
3. 商家可启用已授权的功能开关，但高风险开关受平台控制。
4. 商家可发起定制需求，但是否纳入标准配置、开关或扩展由平台评估。
5. 商家可查看账单和退款对账记录，但不存在提现到平台的流程。

## 5. 商家端核心功能模块

### 5.1 入驻申请与接入资料维护

- 填写主体信息、法人信息、经营分类、品牌信息。
- 上传营业执照、法人身份证、商标证书、授权文件、开户信息。
- 支持保存草稿、提交申请、查看处理进度。
- 支持在资料被驳回后补充并重新提交。

### 5.2 小程序接入

- 录入多个小程序 `AppID`、`AppSecret`、支付商户号、回调地址等信息。
- 支持一键接入检测，自动检查域名、密钥、支付配置、隐私协议配置。
- 支持将不同 `AppID` 绑定到不同店铺。
- 支持查看接入状态：未配置、待验证、已验证、已开通。

### 5.3 一键开通与首版发布

- 通过统一入口完成店铺开通、默认配置初始化、模板绑定和首版发布。
- 自动校验主体资料状态、协议状态、接入配置、装修配置和首批商品数量。
- 校验失败时返回问题清单并支持跳转修复。
- 发布成功后展示模板版本、发布时间、二维码和访问入口。

### 5.4 店铺管理

- 维护店铺名称、Logo、简介、客服信息、退货地址。
- 设置营业状态、服务承诺、发货时效、售后说明。
- 支持同一商家维护多个店铺。
- 支持每个店铺绑定独立小程序。

### 5.5 小程序装修与配置中心

- 配置主题色、导航图标、首页轮播、首页楼层、底部导航文案。
- 配置分享标题、分享图、客服方式、公告内容。
- 支持预览、保存草稿、提交发布、版本回滚。
- 支持按店铺或小程序维度分别维护装修配置。
- 首页模板支持“通用商城版”“生鲜即时零售版”“品牌零售版”三类基线风格。
- 生鲜即时零售版首页采用“服务信息 + 搜索 + 主活动 + 金刚区 + 优惠会场 + 商品楼层 + 推荐流”结构，适合参考盒马鲜生、七鲜这类首页节奏。
- 品牌零售版首页采用“品牌主视觉 + 搜索 + 分类入口 + 专题会场 + 内容楼层 + 推荐流”结构，适合服饰、美妆、家居、母婴、数码等非生鲜商家。
- 商家可配置顶部服务信息，包括门店名、地址文案、最快送达时间、营业状态、配送说明。
- 商家可配置首页金刚区频道，例如水果蔬菜、肉蛋水产、乳品烘焙、酒水零食、熟食快手菜、日用百货。
- 商家可配置优惠券卡片、活动会场、时令推荐、热销榜单、凑单专区、品牌专区等首页楼层。
- 商品楼层应支持大图卡片、双列商品流和横滑商品组，突出商品图、活动价、规格与快速加购。

生鲜类首页建议支持的能力：
- 地址与门店：支持地址选择、当前门店展示、门店切换、配送范围提示。
- 履约信息：支持展示最快送达时间、起送金额、配送费说明、营业状态。
- 搜索能力：支持热搜词、默认搜索词、搜索直达商品或分类会场。
- 频道入口：支持金刚区频道图标、名称、角标、排序、跳转配置。
- 券与权益：支持新人券、满减券、限时券、会员权益入口配置。
- 运营会场：支持主 Banner、次 Banner、时令会场、专题会场、横滑活动卡。
- 商品楼层：支持爆款、排行榜、快手菜、凑单专区、品牌专区、手动精选。
- 商品卡片：支持展示价格、划线价、规格、促销标签、库存状态、销量、加购按钮。
- 状态提示：支持售罄、缺货、休息中、超出配送范围等前台提示。
- 浮层入口：支持购物车悬浮入口、客服悬浮入口、返回顶部。
- 数据统计：支持楼层曝光、点击、加购、券领取、活动点击等数据回流。

非生鲜类首页建议支持的能力：
- 品牌表达：支持品牌头图、品牌文案、品牌故事、主推系列展示。
- 搜索能力：支持热搜词、默认搜索词、搜索直达商品、分类、品牌专区和专题会场。
- 分类与导购：支持核心分类入口、新品专区、爆款专区、系列专区、场景专区。
- 营销权益：支持优惠券、满减、会员权益、积分专区和大促入口。
- 内容楼层：支持图文混排、品牌馆、搭配推荐、种草内容、选购指南。
- 商品卡片：支持展示券后价、会员价、评价数、销量、卖点标签、收藏或加购按钮。
- 专题运营：支持节庆会场、上新会场、联名会场和品牌主题页。
- 用户分层：支持按新客、老客、会员、活动用户展示不同权益入口和推荐内容。
- 浮层入口：支持客服、领券、会员中心、返回顶部等浮层操作。
- 数据统计：支持楼层曝光、点击、收藏、加购、领券、专题跳转等数据回流。

首页模块数据来源规则：
- 分类入口优先绑定商家自有分类树，不直接绑定平台分类模板。
- 商品楼层支持分类、商品组、活动、排行榜、手动选品、品牌专题等数据来源。
- 活动类模块应支持生效时间和失效时间，到时自动上下线。
- 用户分层展示至少支持新客、老客、会员三类规则。
- 多门店商家在生鲜类首页下，服务条和商品楼层应允许随门店切换动态加载。

### 5.6 账号与权限中心

- 主账号创建、启用、停用、重置子账号。
- 维护子账号姓名、手机号、备注、负责业务。
- 分配模块权限、动作权限和数据范围。
- 记录关键操作审计日志。

### 5.7 功能开关中心

- 对已授权能力进行启停管理。
- 支持布尔开关、枚举开关、阈值开关、白名单开关。
- 支持按商家、模板版本、环境和生效时间设置。
- 支持依赖校验和互斥校验。

适合开关化的典型功能：
- 营销能力：优惠券、满减、秒杀、拼团、积分、会员价。
- 交易能力：预约下单、自提、备注、发票申请、限购。
- 展示能力：活动楼层、公告、品牌页、客服悬浮窗。
- 履约能力：物流、自提核销、同城配送、发货时效展示。

### 5.8 定制化申请中心

- 提交业务目标、期望页面、功能说明、期望上线时间。
- 查看需求类型、处理状态、负责人、预计版本。
- 查看处理进度和历史记录。

### 5.9 商品分类管理

- 从平台分类模板初始化商家自己的分类树。
- 从平台分类属性模板初始化分类属性规则。
- 查看模板原始结构和商家当前结构差异。
- 支持新增、编辑、排序、启停、迁移、预览。
- 支持为分类维护属性，包括属性名称、属性类型、是否必填、可选值、是否前台展示、是否参与筛选。
- 支持基于平台属性模板复制后自定义新增、修改、停用、删除、排序。
- 删除分类前校验是否仍有商品挂载。
- 删除属性前校验是否已有商品使用；若已使用，需先处理历史数据或保留属性快照。

### 5.10 商品管理

- 新增、编辑、上下架、删除商品。
- 维护 `SPU/SKU`、图片、视频、详情、库存、价格、运费模板。
- 支持按状态筛选草稿、已上架、已下架、发布失败商品。
- 商品发布时必须绑定当前商家的商品分类。
- 商品编辑时根据分类自动加载当前分类属性，支持录入商品属性值。
- 商品保存后保留商品属性快照，避免后续分类属性变化影响历史商品展示。

### 5.11 商品一键发布

- 支持单商品和多商品批量发布。
- 发布前自动校验商品名称、分类、主图、详情、SKU、价格、库存、配送信息。
- 校验通过则直接上架。
- 校验失败则返回失败原因和修复建议。
- 同一批次允许部分成功、部分失败。

### 5.12 订单管理

- 查看订单列表和详情。
- 发货、打印面单、填写物流单号。
- 对异常订单进行备注。
- 支持订单导出。

### 5.13 售后管理

- 处理退款申请。
- 审核退货申请。
- 上传拒绝理由或协商结果。
- 查询售后历史记录。

### 5.14 账务中心

- 查看账单明细、退款冲减、账期和账单状态。
- 查看退款冲减明细和对账差异说明。
- 平台不抽佣，账务能力以记录和对账为主。

### 5.15 数据看板

- 查看销售额、订单量、访客数、转化率、退款率。
- 查看首页装修点击数据、活动参与数据、配置版本数据。
- 支持按日、周、月查看趋势。

## 6. 商家端关键流程

### 6.1 入驻与开通流程

1. 商家注册账号。
2. 提交主体资料和经营信息。
3. 查看处理进度并补充资料。
4. 录入小程序接入参数。
5. 完成模板选择和基础配置。
6. 通过一键开通完成店铺初始化和首版发布。

### 6.2 定制化申请流程

1. 商家提交需求。
2. 系统记录需求类型和处理状态。
3. 平台评估归属为标准配置、功能开关或审批制扩展。
4. 商家查看进度、预计版本和结果。

### 6.3 商品发布流程

1. 商家创建或编辑商品草稿。
2. 选择当前商家分类并补全商品信息。
3. 系统自动加载分类属性，商家补全必填属性和值。
4. 选择单个或多个商品执行一键发布。
5. 系统执行发布校验。
6. 校验通过的商品直接上架，失败商品保留原状态。

## 7. 商家端状态与规则

### 7.1 账号与权限规则

- 主账号默认全量权限。
- 子账号权限至少包含功能权限和数据范围权限两层。
- 高风险权限建议单独授权，如发布、导出、账号管理。
- 子账号停用后应立即失效，但操作日志必须保留。

### 7.2 商品与发布规则

- 可发布商品状态：草稿、已下架、发布失败后修正完成。
- 不可发布商品状态：已上架、库存锁定中的商品。
- 一键发布失败不得影响同批次其他已通过校验商品。

### 7.3 分类规则

- 商家分类树归属单一商家，不与其他商家共享。
- 商家可调整分类名称、层级、排序和启停状态，但不得突破平台设定的最大层级和禁用词规则。
- 删除分类前必须先迁移或下架关联商品。
- 商家可基于平台属性模板初始化分类属性，也可按规则新增自定义属性。
- 删除或停用分类属性前必须校验是否已有商品使用；若已使用，需保留商品属性快照或先完成迁移。

### 7.4 账务规则

- 平台不抽佣。
- 商家收入不计入平台收入口径。
- 退款后应自动形成账单冲减记录。

## 9. 商家端验收要点

1. 商家可独立完成资料提交、接入配置和进度查询。
2. 商家可独立完成店铺开通、装修配置和首版发布。
3. 商家主账号可创建子账号并分配店铺/小程序权限。
4. 商家可维护自己的分类树和商品数据。
5. 商家可对多个商品执行一键发布并获得明确反馈。
6. 商家可处理订单、售后、账单和基础经营数据查看。
