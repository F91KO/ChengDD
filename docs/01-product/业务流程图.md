# 程哒哒业务流程图

## 1. 文档说明

- 文档用途：基于当前 PRD 输出业务侧主流程图，供产品、研发、设计、实施统一理解业务链路。
- 当前范围：聚焦多商家小程序平台的核心业务，不展开底层技术实现细节。
- 建议用途：需求评审、原型设计、技术方案拆解、排期讨论。

## 2. 平台整体业务总览

```mermaid
flowchart LR
    A["平台配置模板与规则"] --> B["商家入驻与资料提交"]
    B --> C["小程序接入配置"]
    C --> D["选择模板并一键开通"]
    D --> E["商家装修首页与店铺"]
    E --> F["维护分类、属性、商品"]
    F --> G["商品一键发布"]
    G --> H["消费者访问小程序"]
    H --> I["下单、支付、售后"]
    I --> J["商家处理订单与售后"]
    J --> K["平台与商家查看经营数据"]
    A --> L["平台版本发布与模板治理"]
    L --> D
    L --> E
    L --> F
```

## 3. 商家入驻与开通流程

```mermaid
flowchart TD
    A["商家注册账号"] --> B["提交主体资料与经营信息"]
    B --> C["平台处理资料"]
    C --> D{"是否通过"}
    D -- "否" --> E["补充资料或重新提交"]
    E --> B
    D -- "是" --> F["配置小程序 AppID / 支付参数"]
    F --> G["执行接入检测"]
    G --> H{"检测是否通过"}
    H -- "否" --> I["按问题清单修复配置"]
    I --> G
    H -- "是" --> J["选择首页模板与分类模板"]
    J --> K["一键开通并初始化店铺"]
    K --> L["生成默认首页配置与分类树"]
    L --> M["生成首版发布任务"]
    M --> N{"发布是否成功"}
    N -- "否" --> O["查看失败原因并修复后重试"]
    O --> M
    N -- "是" --> P["商家获得小程序访问入口"]
```

## 4. 商家首页装修与发布流程

```mermaid
flowchart TD
    A["商家进入装修中心"] --> B["选择首页风格"]
    B --> C{"首页风格"}
    C -- "生鲜即时零售版" --> D["配置服务条、金刚区、会场、商品楼层"]
    C -- "品牌零售版" --> E["配置品牌主视觉、专题会场、内容楼层"]
    C -- "通用商城版" --> F["配置轮播、分类入口、推荐商品"]
    D --> G["预览首页效果"]
    E --> G
    F --> G
    G --> H{"是否通过校验"}
    H -- "否" --> I["修正缺失项或错误配置"]
    I --> G
    H -- "是" --> J["保存草稿或提交发布"]
    J --> K["生成装修版本"]
    K --> L["发布到目标店铺/小程序"]
    L --> M["前台首页生效"]
```

## 5. 商品分类、属性与商品发布流程

```mermaid
flowchart TD
    A["商家选择平台分类模板"] --> B["初始化商家分类树"]
    B --> C["初始化分类属性模板"]
    C --> D["商家维护自有分类与属性"]
    D --> E["创建或编辑商品"]
    E --> F["选择分类并自动加载分类属性"]
    F --> G["填写商品属性、价格、库存、图文详情"]
    G --> H["保存商品草稿"]
    H --> I["勾选商品执行一键发布"]
    I --> J["系统校验分类、属性、SKU、图片、库存等"]
    J --> K{"是否通过"}
    K -- "否" --> L["返回商品级失败原因"]
    L --> E
    K -- "是" --> M["商品上架"]
    M --> N["前台首页/分类页可见"]
```

## 6. 消费者交易流程

```mermaid
flowchart TD
    A["用户进入商家小程序首页"] --> B["浏览首页活动、分类、商品"]
    B --> C["搜索或进入分类页"]
    C --> D["查看商品详情"]
    D --> E["加入购物车或立即购买"]
    E --> F["提交订单"]
    F --> G["在线支付"]
    G --> H{"支付是否成功"}
    H -- "否" --> I["返回订单页重试支付"]
    I --> G
    H -- "是" --> J["商家接单"]
    J --> K["商家发货或履约"]
    K --> L["用户收货"]
    L --> M{"是否申请售后"}
    M -- "否" --> N["订单完成"]
    M -- "是" --> O["用户提交售后申请"]
    O --> P["商家处理售后"]
    P --> Q["退款或退货退款完成"]
```

## 7. 平台模板治理与版本发布流程

```mermaid
flowchart TD
    A["平台维护首页模板、分类模板、功能开关"] --> B["生成模板版本"]
    B --> C["完成公共回归验证"]
    C --> D["选择目标商家范围"]
    D --> E["执行兼容性校验"]
    E --> F{"是否通过"}
    F -- "否" --> G["输出阻塞项并调整模板或商家配置"]
    G --> E
    F -- "是" --> H["发起灰度发布或全量发布"]
    H --> I["记录发布结果与审计日志"]
    I --> J{"是否需要回滚"}
    J -- "否" --> K["模板版本生效"]
    J -- "是" --> L["执行版本回滚"]
    L --> M["恢复到上一稳定版本"]
```

## 8. 首页模板选择流程

```mermaid
flowchart TD
    A["商家进入首页模板选择"] --> B["查看行业与经营模式"]
    B --> C{"商家类型"}

    E --> G
    F --> G
    G --> H["商家按需调整模块启停与顺序"]
    H --> I["保存首页配置"]
```
