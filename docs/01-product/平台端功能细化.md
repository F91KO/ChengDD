# 程哒哒平台端功能细化说明

## 1. 文档定位

- 文档用途：基于 PRD 拆分平台侧功能，便于产品设计、平台后台规划、研发实施和交付说明。
- 适用对象：产品经理、研发、测试、交付实施、平台管理人员。
- 对应范围：平台交付管理员、平台管理员在后台可见和可操作的功能。

## 2. 平台端定位

1. 平台的核心定位是帮助商家发布小程序、提供商城通用功能，而不是商家业务运营平台。
2. 平台负责模板治理、版本发布、商家接入、分类模板、开关治理、扩展治理、数据查看和交付支撑能力。
3. 平台不承担商家日常经营决策，不做平台佣金、不做提现、不做商家收入归集。
4. 平台可查看商家数据，但重点是交付状态、发布状态、版本状态和基础经营数据。
5. 平台在部署策略上优先采用低成本、标准化方案，并可按项目需要提供 CI/CD 服务。

## 3. 平台端角色

### 3.1 平台交付管理员

- 负责商家接入处理、模板配置、分类模板管理、版本发布和数据查看。
- 负责商家功能授权范围管理。
- 负责标准交付链路中的问题处理和发布支持。

### 3.2 平台管理员

- 负责系统配置、账号权限、审计日志、关键规则管理。
- 负责高风险开关和高风险扩展点的控制。
- 负责平台级安全、权限和治理规则维护。

## 4. 平台端功能边界

1. 平台可处理商家接入资料，但不介入商家日常商品经营管理。
2. 平台提供分类模板和分类属性模板，不强制所有商家使用统一分类结果。
3. 平台提供功能开关和扩展点能力，但不为单商家长期维护独立代码分支。
4. 平台支持版本发布、灰度、回滚和兼容性校验。
5. 平台可查看订单、退款、支付等数据，但以交付治理和状态分析为主。

## 5. 平台端核心功能模块

### 5.1 商家管理

- 查看商家资料、接入状态和当前开通状态。
- 按状态筛选：待处理、资料待补充、已通过、已冻结、已停用。
- 对商家执行冻结、解冻、清退等平台级动作。

### 5.2 分类模板与品牌管理

- 配置平台标准商品分类模板。
- 维护多个行业模板，供商家开通时选用。
- 在分类模板或分类节点下维护属性模板，供商家初始化商品属性结构。
- 维护分类模板对应的资质要求、发布约束和最大层级。
- 管理品牌库和授权资料。

### 5.3 分类模板治理

- 创建、编辑、停用、复制分类模板。
- 维护模板版本、适用行业、推荐商家类型、默认排序规则。
- 维护分类属性模板，包括属性名称、属性类型、必填规则、可选值、排序和受控属性标记。
- 查看模板被采用次数、模板偏离率和商家常见自定义情况。
- 查看属性模板被采用次数、属性偏离率和商家常见自定义属性情况。
- 下发模板升级建议，但不直接覆盖商家现有分类树。

### 5.4 模板与组件管理

- 管理小程序公共模板版本。
- 管理标准装修模块和扩展组件。
- 管理默认配置和适用范围。
- 维护模板版本与商家运行版本映射关系。
- 管理首页模板风格，至少包含“通用商城版”“生鲜即时零售版”“品牌零售版”。
- 为生鲜即时零售版首页提供标准模块库，包括服务条、金刚区、券卡、活动会场、商品楼层、排行榜、品牌专区和推荐流。
- 为品牌零售版首页提供标准模块库，包括品牌头图、专题会场、图文混排、品牌馆、内容楼层、会员权益卡和推荐流。
- 为首页模块提供统一参数协议，保证商家换模板时仍可复用大部分首页配置。
- 支持平台配置首页默认埋点口径，统一统计曝光、点击、加购和活动转化。

### 5.5 商家定制化管理

- 评估商家定制需求。
- 标记需求归类为标准配置、功能开关或审批制扩展。
- 记录评审结论、排期、交付版本和影响范围。
- 推动高频定制需求回收为平台标准能力。

### 5.6 版本发布管理

- 选择模板版本和目标商家。
- 执行兼容性校验、灰度发布、全量发布和回滚。
- 查看每个商家的当前运行版本和发布历史。
- 支持标准商家的一键首版发布和后续一键重发。

### 5.7 数据查看中心

- 查看商户维度的小程序接入状态、发布状态、版本状态。
- 查看商品、订单、退款、访客、支付等汇总数据。
- 按商户、店铺、小程序、时间范围筛选查看数据。
- 查看商品一键发布批次结果和失败原因分布。

### 5.8 内容与文档管理

- 管理公告、协议文档、帮助中心。
- 管理交付说明、接入规范和发布指引。
- 维护商家使用文档和交付实施文档。

### 5.9 权限与审计

- 管理平台 `RBAC` 权限模型。
- 管理角色、菜单、数据权限。
- 支持按商家主体、店铺、小程序、子账号范围进行校验。
- 所有关键操作保留审计日志。

### 5.10 功能开关治理

- 创建、下线、冻结开关定义。
- 维护默认值、风险等级、控制权限和适用范围。
- 向商家授权可见和可用范围。
- 对高风险开关实行平台强控。

平台重点治理对象：
- 支付相关能力。
- 账务校验能力。
- 风控相关能力。
- 会影响交易链路的数据字段和流程节点。

### 5.11 扩展点与扩展组件治理

- 维护扩展点注册信息、支持版本、风险等级和降级方案。
- 审批页面级、交易级、接口级扩展。
- 跟踪扩展组件启用商家、发布时间、失败记录和回滚记录。
- 确保扩展只能在预留点位和标准协议内生效。

### 5.12 数据权限治理

- 功能权限与数据范围分离配置。
- 控制平台主账号、子账号、商家账号的数据可见范围。
- 敏感字段支持脱敏，如金额、手机号、地址。
- 导出和回滚等高风险操作需记录审计日志。

### 5.13 交付支撑能力

- 维护源码交付包、部署包、初始化脚本和版本记录。
- 支持私有化部署场景的环境规划、升级和回滚。
- 提供轻量版、标准版、高可用版交付方案。
- 在满足要求的前提下优先采用低成本部署方案。
- 可按项目需要提供 CI/CD 服务，包括流水线配置、发布自动化、回滚流程和发布记录留痕。
- 支持交付过程中的版本映射、部署说明和环境切换规则。

## 6. 平台端关键流程

### 6.1 商家接入处理流程

1. 平台查看商家提交的主体资料和经营信息。
2. 对资料执行通过、驳回、补充资料处理。
3. 跟踪商家补充和重新提交情况。
4. 在资料处理通过后进入小程序接入和开通阶段。

### 6.2 商家定制化处理流程

1. 平台接收商家定制需求。
2. 评估需求归属为标准配置、功能开关或审批制扩展。
3. 对审批制扩展进行评审、排期、测试和发布。
4. 记录影响范围和交付版本。

### 6.3 模板版本发布流程

1. 平台研发产出模板版本。
2. 平台完成公共回归验证。
3. 交付管理员选择目标商家范围。
4. 系统执行兼容性校验。
5. 平台执行灰度发布、全量发布或紧急回滚。

## 7. 平台端规则与控制点

### 7.1 商家接入控制规则

- 商家资料处理与版本发布是两条不同流程。
- 商家处理通过后才可进入标准开通链路。
- 冻结和停用商家时需保留历史数据和操作记录。

### 7.2 模板与分类治理规则

- 模板升级不得直接覆盖商家已生效配置。
- 分类模板是初始化来源，不是商家线上分类的唯一来源。
- 分类属性模板同样只作为初始化来源，不允许平台直接覆盖商家已生效的分类属性。
- 平台可控制分类层级、禁用词、资质要求和受限品类边界。

### 7.3 功能开关治理规则

- 高风险开关必须由平台强控。
- 商家只能操作已授权且风险等级允许的开关。
- 开关变更需保留变更前后值、生效时间和影响范围。

### 7.4 扩展治理规则

- 所有扩展点必须先注册后使用。
- 扩展必须在标准扩展点内注入，不得改写核心订单、价格、库存和支付逻辑。
- 高风险扩展需由平台管理员或技术负责人审批。

### 7.5 发布治理规则

- 发布前必须完成模板兼容性、开关依赖、扩展组件和基础配置完整性校验。
- 支持按商家灰度发布和按商家回滚。
- 发布失败必须有明确的失败阶段和建议处理动作。

## 9. 平台端验收要点

1. 平台可完成商家接入处理、商家管理和状态维护。
2. 平台可维护分类模板、模板版本、组件和功能开关。
3. 平台可完成商家维度的版本发布、灰度和回滚。
4. 平台可查看商家接入状态、发布状态和基础经营数据。
5. 平台可管理权限、审计日志和敏感数据访问边界。
6. 平台可支撑源码交付和私有化部署场景。
